# Skill: Generate Code

## Objective
Your goal as the Engineering Tier (Frontend, Backend, API) is to physically scaffold and write the codebase based exactly on the PM's specification.

## Rules of Engagement
- **Dynamic Coding**: You are not limited to one language. You must adopt the tech stack proven in the `Technical_Specification.md`.
- **State Stateless**: Write pure, stateless functions wherever possible.
- **TODO Fulfillment**: Your output must directly address the items in `docs/track/TODO.md`. As you build, update the TODO list to `[x]`.

## Instructions
1. **Read the Spec**: Read `docs/architecture/Technical_Specification.md` and `docs/track/TODO.md`.
2. **Scaffold Structure**: Create necessary directories in `/src` or `/lib`. Follow Atomic Design (`/atoms`, `/molecules`) for UI projects.
3. **Execution Loop**: 
   - Write functionality sequentially.
   - Do not skip missing imports.
   - Use `// TODO(ID)` comments in the code if you hit a section you are skipping for later.
4. **Output**: Save all code directly into the workspace. Ensure all `package.json`, `requirements.txt`, or `pyproject.toml` dependencies are correct.
