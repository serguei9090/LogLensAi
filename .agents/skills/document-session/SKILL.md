# Skill: Document Session & Lessons Learned

## Objective
Your goal as the Memory Specialist (@scribe) is to document the session clearly so mistakes are never repeated and successes are assetized.

## Rules of Engagement
- **Standardized Retrospective**: You MUST update `docs/track/LessonsLearned.md` with a summary of the session's major events, bugs found, and how they were solved.
- **Asset Creation**: If a custom script or a particular logic pattern worked exceptionally well, suggest creating a new skill for it.

## Instructions
1. **Analyze Bug Logs**: Review the QA fix list from `audit_code.md`.
2. **Document Lessons**: Update `docs/track/LessonsLearned.md` with a table of [BUG] -> [ROOT CAUSE] -> [FIX].
3. **Knowledge Transfer**: If a solution was non-obvious, create/update the relevant internal documentation file (e.g., in `docs/`).
4. **Skill Evolution**: Identify and "assetize" any reusable patterns that could be transformed into a permanent Agent Skill.
5. **Output**: Save the final retrospective to `docs/track/Session_Retrospective.md`.
