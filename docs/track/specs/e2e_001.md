## The Contract (What)
Validate the entire pipeline: Ingestion (HTTP) -> Extraction (Facets) -> Storage (DuckDB) -> Retrieval (JSON-RPC) -> UI Rendering (Virtual Table).

## Implementation Strategy
Since we cannot run a real GUI in this environment, we used a "Headless E2E" approach:
1. **Sidecar Ingestion**: Created `sidecar/tests/test_full_pipeline.py` which sends logs to the `IngestionServer` and verifies DuckDB storage and JSON-RPC retrieval.
2. **Frontend Integration**: Created `src/__tests__/InvestigationIntegration.test.ts` which uses the real `investigationStore` and mocks the sidecar bridge to verify UI state mapping.
3. **Automated Verification**: Ran both test suites to confirm full pipeline integrity.

## Lessons Learned
- **Port Conflicts**: Backend App automatically starts background services on init. Tests must use `start_ingestion=False` to avoid binding conflicts when testing custom ports.
- **RPC Consistency**: Ensure JSON-RPC return types match what the frontend `callSidecar<T>` expects (Dictionary vs List).
