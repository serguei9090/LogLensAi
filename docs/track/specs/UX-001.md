# UX-001: Sidebar Collapse Functionality

## 🎯 Objective
Improve workspace real estate by allowing users to collapse the main navigation sidebar into a "mini" version that preserves core navigation while maximizing the log viewing area.

## 📐 Design Specification

### 1. States
- **Expanded (Default)**: `w-64` (256px). Full labels, workspace names, and headers visible.
- **Collapsed**: `w-[68px]`. Only icons visible. Labels and headers hidden.

### 2. Interaction Logic
- **Toggle Trigger**: A new button in the sidebar header (next to the logo or replacing it in collapsed mode) or a dedicated toggle at the bottom.
- **Icons**: 
    - Expanded -> Collapsed: `PanelLeftClose`
    - Collapsed -> Expanded: `PanelLeftOpen`
- **Tooltips**: In collapsed mode, all navigation items (workspaces and bottom nav) MUST show a tooltip on hover with their full name/label.
- **Active State**: The active workspace/nav item must remain visually distinct (e.g., emerald accent) even in collapsed mode.

### 3. Visual Components
- **Header**:
    - Expanded: `Terminal` icon + "LogLensAi" text + `PanelLeftClose` toggle.
    - Collapsed: `Terminal` icon (centered) + `PanelLeftOpen` floating toggle (on hover or persistent).
- **Workspace List**:
    - Expanded: `Database` icon + Workspace Name + Action Icons (Rename/Delete) on hover.
    - Collapsed: `Database` icon (centered). No action icons (or only in tooltip/context menu - deferred).
- **Bottom Nav**:
    - Expanded: Icon + Label + "SOON" badge.
    - Collapsed: Icon (centered). Tooltip shows label + "Coming Soon" if applicable.

## 🛠️ Implementation Plan
1. **State Management**: Add `isCollapsed` to a UI-specific store (or local state if not needed elsewhere, but a store is better for persistence).
2. **Layout Adjustment**: 
    - Update `AppLayout` or wherever `Sidebar` is used to handle the dynamic width.
    - Use Tailwind `transition-all` for smooth resizing.
3. **Component Refactor**:
    - Modify `Sidebar.tsx` to conditionally render text and adjust padding/alignment.
    - Wrap icons in `Tooltip` when `isCollapsed` is true.

## ✅ Acceptance Criteria
- [ ] Sidebar collapses/expands smoothly.
- [ ] Active workspace is visible in both states.
- [ ] Tooltips appear for all items in collapsed mode.
- [ ] Layout of the main content (log table) expands to fill the space.
- [ ] "SOON" badges are handled gracefully in collapsed mode.
