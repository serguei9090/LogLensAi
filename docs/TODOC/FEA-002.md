# FEA-002: Dual-Calendar Refactor (Booking Pattern)

## 🎯 Objective
Refactor the `TimeRangePicker` to use a dual-calendar layout (two months side-by-side) to improve UX when selecting date ranges that span across month boundaries. This aligns with standard "booking" systems but is tailored for "Engine Precision" log analysis.

## 🏗️ Architecture
- **Layout**: Popover content will expand horizontally to accommodate two `DayPicker` instances.
- **State Management**:
  - `month`: Still tracks the "left" calendar month.
  - `nextMonth`: Derived from `month + 1`.
- **Navigation Controls**:
  - The single-button header pattern must be preserved but scaled for the dual view.
  - A shared navigation header should control both months (Prev month shifts both left, Next month shifts both right).

## 🎨 UI/UX Design (Engine Precision)
- **Obsidian Theme**: Maintain dark backgrounds and tactical green/amber accents.
- **Selection Preview**: The integrated preview area at the bottom must remain and update in real-time as users interact with either calendar.
- **Connectivity**: The range selection span must visually flow across the gap between the two months.

## 📋 Implementation Steps
1.  **Modify `TimeRangePicker.tsx`**:
    - Update the CSS grid/flex layout of the calendar section to `display: grid; grid-template-columns: 1fr 1fr;`.
    - Render two calendars (or a single `DayPicker` with `numberOfMonths={2}`).
2.  **Harmonize Navigation**:
    - Ensure `CalendarHeader` correctly shifts both months.
    - Update `view` toggling (Months/Years) to handle the expanded layout gracefully (possibly only showing the selector for the current active month or a global year picker).
3.  **Refine "Selection Preview"**:
    - Ensure formatting includes years (already implemented, but verify alignment with dual layout).
4.  **Verification**:
    - Test ranges spanning multiple months (e.g., Feb 28 -> Mar 1).
    - Ensure "Engine Precision" HH:MM:SS controls remain synchronized.

## 🛠️ Constraints
- **Library**: `react-day-picker` (v9.x) supports `numberOfMonths={2}`.
- **Atomic Design**: Keep components modular as per `src/components/atoms`, `molecules`, etc.
